# SEO Audit, elevatecareerhub.com (baseline)

Date: 2026-06-25. Scope: the Vite + zero-JS SSG build (the new site), head/meta,
structured data, social cards, icons, sitemap/robots, crawlability. This is the
baseline the weekly routine (`seo-weekly`) measures against once GSC/DataForSEO
credentials are connected.

## Fixed in this pass

- **Social preview was a blank white box (critical).** `og:image` / `twitter:image`
  pointed at `/assets/wp-content/uploads/2024/01/2.png`, which is a white logo on a
  transparent background, so it rendered empty on every share (WhatsApp, LinkedIn,
  Facebook, X, iMessage, Slack). Replaced with a built 1200x630 card,
  `/assets/og/og-default.png` (the Elevate signature on brand blue + tagline), served
  from the repo so it deploys with the site. Added `og:image:width/height/type/alt`
  and `twitter:image:alt`, which WhatsApp and LinkedIn use to render reliably.
- **Site icon refreshed.** Swapped the generic block-"E" for the Elevate signature
  badge across `favicon.ico` (16/32/48/64), `apple-touch-icon.png` (180, opaque),
  and the PWA `icon-192/512`. Removed the stale `favicon.svg` (old mark) and its
  `<link>`. Note: at 16 to 32px the script is faint by nature; the white disc keeps
  the mark recognizable in a browser tab.

## Still open (prioritized)

1. **No `llms.txt`** for AI crawlers. GEO gap. Queued as `tech-llms-txt` in
   `seo/roadmap.yml`: add a writer alongside the sitemap in
   `scripts/build-priority-routes.mjs`.
2. **One shared OG card for every page.** Fine as a baseline, but blog posts and
   service pages convert better with per-page cards. Queued as `tech-og-images-blog`.
3. **Article JSON-LD** has no `dateModified` and points its `image` at the shared OG
   card. Add `dateModified` and, once per-article cards exist, a per-article image.
4. **`<html lang="en-US">`** while the brand is Ghana-first + diaspora. Minor
   relevance signal; consider `en-GH` or leaving as-is.
5. **Vestigial WordPress snapshots** in `public/snapshots/` still embed the old blank
   `2.png` (30 files). They are no longer served for any route (every route has a
   priority page), so this is dead weight, not a live issue. Tracked for the optional
   SPA/snapshot removal (route-migration-plan Phase 5).

## Already strong (leave alone)

- Per-page `<title>` + meta description; canonical on every page; `robots`
  index/follow with `noindex` on the 404 and staging.
- Full JSON-LD `@graph`: Organization, WebSite, BreadcrumbList, Article, Course,
  FAQPage, assembled per route and merged.
- `sitemap.xml` generated from the indexable route list (cannot drift), referenced by
  `robots.txt`; commerce shells correctly disallowed.
- `twitter:card=summary_large_image`, `theme-color`, web manifest, apple-touch icon.
- Zero-JS static SSG with ~0.3 to 0.4s LCP on Regular 3G. Strong Core Web Vitals are
  a real, durable ranking asset; protect this (no hydration on content pages).

## How to verify the social-card fix after deploy

- Re-scrape the URL in each platform's debugger (they cache aggressively):
  Facebook Sharing Debugger, LinkedIn Post Inspector, X Card Validator. WhatsApp
  refreshes from the OG tags once the platforms re-fetch.
- Confirm `https://elevatecareerhub.com/assets/og/og-default.png` returns 200 (the
  card is a new path, so the 1-year static cache has nothing stale to serve).
