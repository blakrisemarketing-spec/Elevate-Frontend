# Content Briefs

One brief per article, named `<slug>.md`, written just before drafting. A brief is
the bridge between the roadmap line and the published post: it fixes the angle so the
draft does not wander.

## What a brief contains

- **Primary keyword** + search intent (informational / commercial / local).
- **Secondary keywords** the article should also cover naturally.
- **The angle** — the Insider Friend take, in one sentence. What do we say that the
  generic listicles ranking today do not?
- **H2 outline** — each H2 with the one self-contained, citable claim sentence it will
  open with (this is what AI engines lift).
- **Internal links** — which existing articles and which money pages
  (`/career-services/`, `/educational-services/`, `/diy-products/`,
  `/get-into-grad-school-bootcamp/`, `/contact-us/`) this post links to, woven as
  inline prose (the blog model has no link block yet, see roadmap
  `tech-blog-link-faq-blocks`).
- **The CTA** — the approved closing framing ("Book a free chat" / "Tell us the goal
  and we'll map the path").
- **Proof points** — any real founder story or canonical stat to use (6+ years, 96%,
  2,000+, Manchester full-ride, fintech-via-cold-outreach). Never invent one.

## Accuracy reminder

Quote prices only from `ech-service-catalog` / `src/checkout/catalog.ts`. Keep the
honest caveat on anything outside our control (final admission and scholarship funding
sit with the providers). No fabricated testimonials.
